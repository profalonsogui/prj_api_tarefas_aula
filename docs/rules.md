# Rules do Projeto

## O que são Rules?

Rules são regras e padrões definidos para o projeto.

Elas servem para:
- manter organização
- evitar bagunça
- padronizar o desenvolvimento
- facilitar manutenção
- melhorar trabalho em equipe

Mesmo projetos simples precisam de regras básicas.

Neste projeto, as rules serão simples e focadas no aprendizado.

---

# Rules Base - Projeto API de Tarefas

## 1. Entender antes de copiar

Nunca copiar código sem entender.

Antes de usar qualquer código:
- ler
- testar
- entender o que faz

---

## 2. Fazer uma funcionalidade por vez

Não tentar desenvolver tudo junto.

Ordem:
1. banco de dados
2. conexão
3. rotas
4. validações
5. melhorias

---

## 3. Manter código organizado

Evitar:
- código bagunçado
- funções gigantes
- arquivos desorganizados

---

## 4. Usar nomes claros

Usar nomes simples e fáceis de entender.
